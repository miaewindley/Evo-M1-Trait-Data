# Audit — mapped vs available volume structures (Part I.C)

Generated by `_audit_missing_structures.R`. For every reference that is in the
volume merge (has a `standardized_term_by_reference/*_standardized_terms.csv`)
**and** a `*_definitions.csv`, every volume-bearing column defined in the paper
is compared against what is actually mapped into the merge.

Output: `_audit_missing_structures.csv` — one row per defined volume column with
`mapped` (TRUE/FALSE), the `standardized_term` it maps to (if any), and a
`category`:

- `mapped` — already in the merge.
- `candidate_missing` — a distinct structure defined in the paper but not mapped.
- `laterality_variant_of_mapped` — a one-side (L/R) column whose structure is
  already mapped (usually via a combined both-sides term); not a new structure.
- `stat_or_nonbrain` — SEM/SD/percent columns or non-brain residuals
  (meninges/hypophysis/nerves); never merged.

## Conclusion

The only true missing-structure case that warrants action this round is the
**Barger amygdala subnuclei** expansion (Part I.B). The remaining candidates are
either laterality sub-parts of already-merged structures, optional finer splits,
size covariates, or definition-vs-data naming artifacts.
